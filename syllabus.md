## Course Description

This course introduces advanced programming concepts and tools commonly used in modern scientific research and computational environments. The focus is on **practical, real-world workflows**, collaborative development, and performance-aware computing, with examples drawn from current student projects and scientific use cases.

Students will actively contribute to the course material using **GitHub issues and pull requests**, simulating real open-source and research software development practices.

---

## Learning Objectives

By the end of the course, students will be able to:

- Compare and select appropriate programming languages for scientific problems
- Use version control systems effectively in collaborative environments
- Apply modern development workflows including continuous integration
- Understand virtualization and containerization for reproducible research
- Share and structure scientific software as reusable components
- Identify and apply basic strategies for performance optimization

---

## Course Structure and Topics

### 1. Programming Languages for Scientific Computing

An overview of the programming language landscape in scientific research, with emphasis on trade-offs between usability, performance, and ecosystem support.

**Topics include:**
- Python as a general-purpose scientific language and its ecosystem
- C++ and Fortran for high-performance and large-scale simulations
- MATLAB in engineering, control systems, and signal processing
- Emerging and alternative languages: Go, R, and Julia

**Practical activities:**
- Comparative examples across languages
- Discussion of real research use cases

---

### 2. Version Control and Software Development Workflows

Introduction to modern collaborative software development practices.

**Topics include:**
- Distributed version control with Git
- Repository organization and branching strategies
- Automatic changelog generation
- Continuous Integration / Continuous Development (CI/CD)
- Automation in scientific code development

**Practical activities:**
- Collaborative work via GitHub
- Issue-driven development and peer review

---

### 3. Virtualization and Reproducibility

Tools and techniques for reproducible computational environments.

**Topics include:**
- Lightweight virtualization with containers (Docker)
- Full virtualization using hypervisors (e.g. VirtualBox)
- Reproducibility and portability of scientific workflows

**Practical activities:**
- Running and modifying containerized scientific applications

---

### 4. Collaboration and Software Sharing

Best practices for sharing scientific code and data.

**Topics include:**
- Shared repositories and standard file formats
- Structuring software as reusable libraries
- Python modules as an example of software packaging
- Open-source practices in research

**Practical activities:**
- Publishing and documenting reusable code components

---

### 5. Optimization and Performance

Introduction to performance-aware programming and computational acceleration.

**Topics include:**
- Identifying performance bottlenecks
- GPU and FPGA acceleration concepts
- High-Throughput Computing (HTC)
- Job-based computation using HTCondor

**Practical activities:**
- Performance profiling
- Discussion of scalability strategies

---

## Teaching Methodology

- Lectures combined with hands-on sessions
- Collaborative development using GitHub
- Active use of issues for discussion, planning, and documentation
- Peer review of contributions via pull requests

---

## Assessment

Assessment is based on **continuous participation and contribution**, including:

- Opening and discussing GitHub issues
- Submitting pull requests that improve code or documentation
- Reviewing and commenting on peer contributions
- Quality, clarity, and correctness of technical work

---

## Prerequisites

- Basic programming experience
- Familiarity with command-line tools
- Motivation to work collaboratively

---

## Recommended Tools

- Git and GitHub
- Python and scientific libraries
- Docker
- Virtualization tools as needed

---

## Notes

The course content is intentionally **evolving**. Students are encouraged to propose improvements, examples, and corrections throughout the semester.
